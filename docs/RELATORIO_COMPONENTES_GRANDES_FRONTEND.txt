3. PADRAO RECOMENDADO PARA QUEBRAR ARQUIVOS
===========================================

Quando uma tela passar de 400 linhas, avaliar separacao em:

- components/
  Partes visuais reutilizaveis ou secoes da pagina.

- hooks/
  Estado, carregamento e acoes assincornas.

- services/
  Regras de chamada ou transformacao especificas do modulo.

- viewModels/
  Preparacao de dados para exibicao.

- constants/
  Listas fixas, opcoes de filtro e mapas de status.


4. CRITERIO PARA MARCAR COMO CONCLUIDO
======================================

Uma refatoracao pode ser considerada concluida quando:

- O arquivo principal da pagina ficar preferencialmente abaixo de 350 linhas.
- O comportamento visual continuar igual.
- O build do front passar.
- Nenhuma chamada HTTP for duplicada desnecessariamente.
- O estado da tela continuar previsivel.


5. COMANDO USADO PARA MEDICAO
=============================

PowerShell:

```text
Get-ChildItem -Path frontend/src -Recurse -Include *.jsx,*.js |
  ForEach-Object {
    $count = (Get-Content -LiteralPath $_.FullName | Measure-Object -Line).Lines
  }
```
